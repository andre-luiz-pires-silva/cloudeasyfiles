# CloudEasyFiles 0.2.0 Release Notes

`0.2.0` is the first multi-provider release of CloudEasyFiles, adding full Azure Blob Storage support and formal GitHub-based release publishing.

## Highlights

- Azure connection management with secure credential storage
- Azure container and blob browsing with incremental loading
- Azure upload, tracked download, folder creation, delete, tier change, and archive rehydration flows
- Provider-aware configuration for upload tier and listing page size
- Linux and Windows installers published through tag-based GitHub Releases
- Repository quality and collaboration baseline with CI, templates, and CODEOWNERS

## Known Limitations

- Quality gates are still centered on `npm run check` and `cargo check`, not a broad automated test suite
- macOS installers are not part of the current release pipeline
- Transfer workflows do not yet provide a local queue or advanced concurrency controls

## Next Milestone

The next work cycle should focus on deepening quality automation, polishing public presentation, and expanding post-release UX hardening rather than opening another provider immediately.
